## ğŸ¯ Key Files Explained

### Core Application Files

#### `cms_app/models.py` (459 lines)
Defines all database models:
- **SiteConfiguration**: Global site settings
- **Page**: Website pages
- **Section**: Page sections
- **ContentBlock**: Content blocks within sections
- **MenuItem**: Navigation menu items
- **Media**: Media library
- **GalleryImage**: Gallery images for blocks

#### `cms_app/admin.py` (245 lines)
Configures Django admin interface:
- Inline editors for nested content
- Sortable lists with drag-and-drop
- Custom list displays and filters
- Rich text editor integration
- Image preview thumbnails

#### `cms_app/views.py` (96 lines)
View logic:
- `HomePageView`: Renders homepage
- `PageDetailView`: Renders individual pages
- `MediaLibraryView`: Media browser
- `robots_txt`: SEO robots.txt

#### `cms_app/api/serializers.py` (188 lines)
REST API serializers:
- Nested serializers for complex data
- Read-only endpoints
- Filtering and pagination support

### Template Files

#### `templates/cms_app/base.html`
Master template with:
- SEO meta tags (title, description, OpenGraph)
- Bootstrap 5 CSS/JS
- Dynamic theming CSS variables
- Navigation and footer includes
- Google Analytics integration

#### `templates/cms_app/includes/section.html`
Section renderer that handles:
- 12+ different section types
- Dynamic block rendering
- Responsive layouts
- Bootstrap components (carousel, accordion, etc.)

#### `templatetags/cms_tags.py`
Custom template tags:
- `render_content_block`: Renders blocks by type
- `render_section`: Renders complete sections
- `section_style`: Generates CSS for sections
- `block_style`: Generates CSS for blocks

### Static Files

#### `static/css/custom.css` (430 lines)
Custom styles for:
- Navigation animations
- Section styling
- Content block types
- Gallery effects
- Responsive design
- Print styles

#### `static/js/custom.js` (340 lines)
JavaScript features:
- Smooth scrolling
- Scroll-to-top button
- Lazy loading images
- Gallery lightbox
- Scroll animations
- Form validation helpers

## ğŸ“¦ Important Directories

### `/media/` - User Uploads
**Auto-created on upload**:
- Organized by date: `/uploads/2024/01/image.jpg`
- Type-specific folders: `/branding/`, `/galleries/`
- **NOT in version control** (see .gitignore)

### `/static/` - Application Static Files
**Source files**:
- CSS, JavaScript, images
- Part of application code
- **In version control**

### `/staticfiles/` - Collected Static Files
**Generated by `collectstatic`**:
- Combined from all apps
- Compressed by WhiteNoise
- **NOT in version control**

### `/docs/` - Documentation
**Markdown documentation**:
- Setup guides
- API reference
- Deployment instructions
- Architecture diagrams
